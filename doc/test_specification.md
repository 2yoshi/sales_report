## 1. テスト方針

### 1.1 テストの目的

- 要件定義・画面定義・API仕様に基づき、システムが正しく動作することを確認する
- ロールごとのアクセス制御が意図通り機能していることを確認する
- 入力バリデーションによる不正データの排除を確認する
- 主要な業務フローが正常に完結できることを確認する

### 1.2 テスト種別

| 種別 | 内容 |
|------|------|
| 機能テスト | 各機能が仕様通りに動作することを確認する |
| 画面テスト | 画面定義書に基づきUI・遷移・表示を確認する |
| APIテスト | APIエンドポイントのリクエスト・レスポンスを確認する |
| 権限テスト | ロールごとの操作可否が正しいことを確認する |
| バリデーションテスト | 入力値の境界値・異常値を確認する |
| 非機能テスト | パフォーマンス・セキュリティ等を確認する |

### 1.3 合否判定基準

| 判定 | 基準 |
|------|------|
| OK | 期待結果と一致する |
| NG | 期待結果と異なる（バグとして起票する） |
| N/A | 当該環境・条件では実施不要 |
